Q1. Three positive numbers are triangular if none of them is greater than or equal to the sum of the other two. (Three triangular numbers could be the side lengths of a triangle.) Implement the function triangular that takes three positive numbers as arguments and returns True if they are triangular, or False if they are not:

def triangular(x, y, z):
    """Return whether x, y, and z are triangular.

    >>> triangular(3, 4, 5)
    True
    >>> triangular(3, 14, 5) # 14 is greater than 3 + 5
    False
    >>> triangular(7.5, 3.5, 4) # 7.5 is equal to 3.5 + 4
    False
    >>> result = triangular(5, 4, 3) # Return, don't print
    >>> result
    True
    >>> triangles = [(1, 2, 2), (2, 1, 2), (2, 2, 1), (2, 2, 2), (4, 3, 5),
    ... (3, 5, 4), (5, 3, 4), (4, 5, 3), (5, 4, 3)]
    >>> [triangular(*t) for t in triangles].count(False)
    0
    >>> lines = [(1, 2, 3), (2, 1, 3), (1, 3, 2), (3, 1, 2), (2, 3, 1),
    ... (3, 2, 1)]
    >>> [triangular(*t) for t in lines].count(True)
    0
    >>> other = [(0.1, 0.1, 1), (0.1, 1, 0.1), (1, 0.1, 0.1)]
    >>> [triangular(*t) for t in other].count(True)
    0
    """
    return x + y + z > 2 * max(x, y, z)

Note

There are many solutions to this problem. The solution above follows from the observation that the sum of the sides of a triangle is always greater than twice its longest side.

Q2. Implement the function next_square that takes as input a positive integer and returns the smallest perfect square greater than its argument. A perfect square is any integer multiplied by itself:

def next_square(x):
    """Return the smallest perfect square greater than x.

    >>> next_square(10)
    16
    >>> next_square(39)
    49
    >>> next_square(100)
    121
    >>> result = next_square(2) # Return, don't print
    >>> result
    4
    >>> cases = [(1, 4), (2, 4), (3, 4), (4, 9), (8, 9), (9, 16), (15, 16),
    ... (16, 25), (24, 25), (25, 36), (26, 36)]
    >>> [next_square(s) == t for s, t in cases].count(False)
    0
    """
    k = 1
    while k * k <= x:
        k = k + 1
    return k * k

Hint: If you're stuck, try writing a function that prints out the first 5 perfect squares using a while statement: 1, 4, 9, 16, 25. Then, adapt that while statement to this question by changing the header.

Note

We iterate over perfect squares until finding one larger than the input. This solution is inefficient, but an efficient solution requires taking a square root, which we will discuss shortly in the class.

Q3. Implement the function digit_span that takes as input a positive integer and returns the difference between its largest digit and its smallest digit. For example, the digits of 2013 are 2, 1, 0, and 3; the difference between 3 and 0 is 3. The digits of 75 are 7 and 5; the difference between 7 and 5 is 2. The return value will never be negative:

def digit_span(x):
    """Return the difference between x's largest and smallest digits.

    >>> digit_span(2013) # 3 - 0 = 3
    3
    >>> digit_span(75) # 7 - 5 = 2
    2
    >>> digit_span(2345678765432) # 8 - 2 = 6
    6
    >>> result = digit_span(6473465) # Return, don't print
    >>> result
    4
    >>> cases = [(1084, 8), (3105, 5), (7000, 7), (174, 6), (714, 6),
    ... (147, 6), (121, 1), (33553355335544664466446644, 3)]
    >>> [digit_span(s) == t for s, t in cases].count(False)
    0
    """
    largest, smallest = x % 10, x % 10
    x = x // 10
    while x > 0:
        largest = max(largest, x % 10)
        smallest = min(smallest, x % 10)
        x = x // 10
    return largest - smallest

Hint: You can use // and % to separate a positive integer into its one's digit and the rest of its digits.

Note

We find the largest and smallest digits, initializing with the one's digit. In each iteration of the while statement, we update largest and smallest, then remove the current one's digit of the remaining digits.
